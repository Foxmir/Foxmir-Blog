title: '强化学习中何时需要处理奖励尺度：判断依据与常用方法'
date: '2026-05-26'
---

* 对于常见取值的折扣因子 $\gamma$ 而言，无论连续动作，还是离散动作：**单步奖励的绝对量级过大** 或 **单步奖励的方差剧烈**，只要满足其一，**就推荐进行奖励的尺度处理**

---

## 原因

* **奖励尺度会直接影响 return、V/Q target、TD error 或策略更新信号的数值尺度，进而影响损失函数和梯度更新的稳定性。** 如果奖励尺度过大或波动过剧烈，就更容易带来 value loss 过大、梯度不稳定、训练震荡等问题。

* 但这不是一个强制规则。在简单任务中，即使不做奖励尺度的处理，训练效果也可能很好。比如利用其他手段压住数值问题，例如：
	- 调整 learning rate
	- gradient/value/reward clipping
	- advantage normalization
	- 合理的网络初始化
	- optimizer 的自适应缩放

---

## 前置知识点：从单步奖励到 V/Q 目标尺度

* 我们真正关心的是**训练中V/Q的准确性与稳定性**，我们需要从这个角度出发判断其他变量对我们的意义。

* **$\gamma$ 的作用之一，是控制未来奖励在当前价值中的权重**
	* 在无限时域问题中，如果单步奖励有界，且 $\gamma < 1$，那么折扣回报会有一个有限上界，不会因为时间无限增长而无限变大。
	* 在有限时长 episode 中，回报本身是有限项求和，不存在无限时域下的收敛性问题，但回报的数值尺度也仍然会受到三个因素共同影响：
			1. 单步 reward 的量级
			2. episode 长度
			3. 折扣因子 $\gamma$ 

* **单步奖励、episode 长度和 $\gamma$ 共同决定了 V/Q target 的数值量级**
	* **数学原理举例说明**：当单步奖励恒为 $r=1$，且 $\gamma=0.99$ 时
		* 如果回合是无限时间步，$V$ 值的理论极限是无穷等比数列求和：$V_{max} = \frac{r}{1 - \gamma} = \frac{1}{1 - 0.99} = 100$
		* 如果回合是有限时间步，则V值只是这个无穷等比数列的截断版本；回合越长，越接近该上界。
		* 注意：从数学上讲，如果单步奖励是无上限的，则该公式不合适，但处理逻辑一致，工程判断上仍然回到“单步奖励量级”和“奖励波动”这两个维度。

* **情景推演**：
	1. **单步奖励不大，但单局原始累计奖励很大**：这可能只是因为回合很长。比如每步 $+1$，一局原始累计 $10000$ 分，但在 $\gamma=0.99$ 下，V/Q target 不会因此变成 $10000$，而是大约被限制在 $100$ 这个量级。因此，这对于V/Q的计算并不带来极端威胁。
	2. **单步奖励本身很大**：例如每步奖励是 $100$，$\gamma=0.99$，那么 V/Q target 的量级可能接近 $10000$。可能带来 value/Q 目标过大、loss 过大、梯度尺度过大的问题，值得进行奖励尺度处理。
	3. **单步奖励均值不大，但偶尔出现很大的奖励或惩罚**：这会让某些 V/Q 突然变大，带来 value/Q 目标波动，难以训练等问题，同样值得进行奖励尺度处理。

---

## 方法

### 方法1：按单步奖励进行 Z-score 标准化

* 底层公式是 **Z-score 标准化**，口语里也常被叫作“奖励归一化”：

$$
r_t' = \frac{r_t - \mu_r}{\sqrt{\sigma_r^2+\epsilon}}
$$

* 和传统静态数据集不同，RL 中的 reward 是动态采样产生的，因此通常使用 **Welford 在线算法** 来维护运行统计量：
	* 不保存所有历史 reward；
	* 只维护当前均值 $\mu_r$、方差 $\sigma_r^2$ 和样本数 $N$；
	* 每收集到新的 batch / rollout 时，用**递推公式更新统计量**。

* 实现上通常遵循：
	* 统计量全局累加；
	* 只在**采样阶段**更新统计量；
	* 训练阶段使用已经冻结的 $\mu_r,\sigma_r^2$，不要一边训练一边继续更新。（比如防止PPO那种采集一批数据，训练多次，如果训练多次都在更新奖励归一化的参数，则显然是不对的。）

* **问题**：这种方法因为公式里的减均值操作，会改变奖励的 0 点。使得归一化后的奖励的零点语义发生变化。

### 方法2：按折扣回报尺度进行奖励缩放

* 这种方法先估计“折扣累计量”的尺度，再用这个尺度去缩放当前 reward。

1. 算法**维护一个前向递推的折扣累计量**，例如：$\tilde R_t = \gamma \tilde R_{t-1} + r_t$ 
2. **再对 $\tilde R_t$ 维护运行方差 $\operatorname{Var}(\tilde R)$，然后用它缩放当前奖励**：$$r_t^{\text{scaled}} = \frac{r_t}{\sqrt{\operatorname{Var}(\tilde R)+\epsilon}}$$

* 如果有多个并行环境，则每个环境各自维护一个 $\tilde R_t$。
* episode 结束时，需要把对应环境的 $\tilde R_t$ 清零。
	* done 那一步的 reward 仍然要先用于更新 $\tilde R_t$ 和统计量，处理完该步数据后再清零
* **注意**：这里的 $\tilde R_t$ 和 GAE / return 里的符号可能看起来相似，但含义不同。它不是训练目标本身，而是用来估计“折扣回报尺度”的运行统计量。

* 这种方法通常**只除以标准差，不减均值**，所以更准确地说是奖励缩放，而不是 Z-score 标准化；它会缩放 reward 的数值尺度，但不会改变 reward 的 0 点语义。

### 其他奖励尺度处理方式

* **奖励裁剪 reward clipping**：把 reward 限制在固定范围内，例如 $[-1, 1]$。优点是能强行压住极端值；缺点是会改变不同奖励之间的相对大小，可能改变任务语义。
* **常数缩放 constant scaling**：直接令 $r' = r / c$，例如把所有 reward 除以 10、100 或 1000。实现简单，语义改动小，但缩放系数需要人工选择。
* **非线性变换**：例如 tanh 函数等，用来压缩特别大的奖励。适合奖励呈指数增长或重尾分布的情况，但会明显改变奖励形状。
* **advantage normalization**：在 PPO / A2C 等策略梯度方法中常用，归一化的是 advantage，而不是 reward 本身；它能稳定 policy update，但不能直接改变 V/Q target 的原始尺度。
* **value / return normalization**：对 return 或 value target 做归一化，例如 PopArt 一类方法；它处理的是价值目标尺度，而不是直接改 reward。
